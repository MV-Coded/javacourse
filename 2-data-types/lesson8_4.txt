Класс IntegerCache — это внутренний статический вложенный класс в Integer, который кэширует экземпляры Integer для значений в диапазоне от -128 до 127 (включительно). 
Это сделано для повышения производительности и экономии памяти, так как небольшие целые числа часто используются в программах.
При автоупаковке (boxing) значений из этого диапазона JVM возвращает один и тот же объект из кэша, а не создаёт новый, что позволяет безопасно использовать оператор == для сравнения только в пределах этого диапазона.
ВЫВОД ПРОГРАММЫ:
a1==i1 true        // Сравнение Integer и int: a1 распаковывается в int, сравниваются значения (128 == 128) → true
b1==i1 true        // Аналогично: b1 распаковывается → 128 == 128 → true
a1==b1 false       // Сравнение двух объектов Integer: оба созданы при значении 128 → выходят за пределы кэша [-128, 127], создаются разные объекты → ссылки разные → false
a1.equals(i1) -> true   // equals сравнивает значения (после автоматического боксинга i1 → Integer), значения равны → true
b1.equals(i1) -> true   // Аналогично → true
a1.equals(b1) -> true   // equals сравнивает значения (128 == 128) → true

a2==i2 true        // Сравнение Integer и int: a2 распаковывается → 127 == 127 → true
b2==i2 true        // Аналогично → true
a2==b2 true        // Оба значения 127 попадают в кэш IntegerCache, поэтому a2 и b2 ссылаются на один и тот же объект → true
a2.equals(i2) -> true   // equals сравнивает значения → true
b2.equals(i2) -> true   // Аналогично → true
a2.equals(b2) -> true   // equals → значения равны → true
